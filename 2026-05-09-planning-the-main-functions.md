After choosing our final topic, we needed to think about the actual functions of the website. I found this step quite important because saying “we want to build a social website” is still too general. We need to know what users can actually do on the website.

The first main function is the homepage or community feed. When users open the website, they should be able to see recent posts from other users. These posts could include daily life, photos, or short tips. I think the homepage should be simple, because it is the first page users see. If the homepage is too confusing, users may not want to continue using the website.

The second function is creating a post. Users should be able to write something and publish it. At first, we thought about allowing video posts, but we decided this was too hard for the prototype. It would also make the website more complicated. So we decided to focus on text posts and maybe image posts. This is more realistic for our current skill level.

The third function is comments and likes. These functions are small, but they are important for social interaction. If users only post but nobody can respond, the website will feel less social. Comments can help users start conversations, and likes can be a quick way to show support.

The fourth function is the life tips section. This section may look similar to posts, but the content is more practical. For example, a user might post a cooking tip or a shopping tip. We may use categories to separate normal daily posts from life tips. This can help users find useful information more easily.

The fifth function is the coupon section. Users can browse available coupons and maybe save them. For the prototype, we do not need to connect to real supermarkets. We can create sample coupon data to show how the function would work. This keeps the function possible to build.

When planning these functions, I also started to think about the database. We may need data for users, posts, comments, likes, tips, coupons, and saved coupons. This helped me understand why planning is important before coding. If we do not know what data we need, it will be harder to build the website later.

Overall, I think our best strategy is to build fewer functions but make them clear and complete.
